LibraryCatalog - Classe Principale

La classe principale qui gère l'état complet de la bibliothèque.

LibraryCatalog.empty

Crée un nouveau catalogue vide.

val catalog = LibraryCatalog.empty

addBook(book: Book): LibraryCatalog

Ajoute un livre au catalogue.

Exemple :
val book = Book( ISBN.safe("978-0134685991"), "Effective Java", List("Joshua Bloch"), 2017, "Programming", true ) val newCatalog = catalog.addBook(book)

addUser(user: User): LibraryCatalog

Ajoute un utilisateur au catalogue.

Exemple :
val user = User.Student( UserID.random(), "Alice Dupont", "Informatique", "password123" ) val newCatalog = catalog.addUser(user)

loanBook(isbn: ISBN, userId: UserID): LibraryCatalog

Emprunte un livre pour un utilisateur.

Exemple :
val loanResult = catalog.loanBook(isbn, userId)

Book - Modèle de Livre

Représente un livre dans la bibliothèque avec validation ISBN.

Constructeur

case class Book( isbn: ISBN, title: String, authors: List[String], publicationYear: Int, genre: String, isAvailable: Boolean )
Création d'un livre :
val book = Book( ISBN.safe("978-0134685991"), // Validation automatique "Effective Java", List("Joshua Bloch"), 2017, "Programming", true )

User - Hiérarchie d'Utilisateurs

Système d'utilisateurs avec trois types : Student, Faculty, Librarian.

User.Student

Étudiant avec limite de 5 livres, 30 jours de prêt.

User.Student(userId, name, department, password)

User.Faculty

Faculté avec limite de 10 livres, 90 jours de prêt.

User.Faculty(userId, name, department, password)

User.Librarian

Bibliothécaire avec accès illimité.

User.Librarian(userId, name, password)

SearchOperations - Recherche Avancée

Opérations de recherche avec prédicats composables.

Prédicats de Base

import SearchOperations.* // Recherche par titre Predicates.byTitle("Java") // Recherche par auteur Predicates.byAuthor("Bloch") // Recherche par genre Predicates.byGenre("Programming")

Combinaison de Prédicats

Recherche complexe :
val complexSearch = Predicates.byGenre("Programming") && Predicates.byAuthor("Bloch") val results = catalog.searchBooks(complexSearch)

Types - Types Opaques

Types sécurisés pour éviter les erreurs de programmation.

ISBN

// Création sécurisée val isbn = ISBN.safe("978-0134685991") // Accès à la valeur val stringValue = isbn.value

UserID

// Génération aléatoire val userId = UserID.random() // Depuis UUID existant val userId2 = UserID(UUID.randomUUID())

Exemples d'Usage Complets

Scénario complet :
// 1. Créer un catalogue vide val catalog = LibraryCatalog.empty // 2. Ajouter des livres val book1 = Book( ISBN.safe("978-0134685991"), "Effective Java", List("Joshua Bloch"), 2017, "Programming", true ) val catalogWithBooks = catalog.addBook(book1) // 3. Ajouter des utilisateurs val student = User.Student( UserID.random(), "Alice Dupont", "Informatique", "password123" ) val catalogWithUsers = catalogWithBooks.addUser(student) // 4. Emprunter un livre val finalCatalog = catalogWithUsers.loanBook( ISBN.safe("978-0134685991"), student.id ) // 5. Rechercher des livres import SearchOperations.* val programmingBooks = finalCatalog.searchBooks( Predicates.byGenre("Programming") )

🛠️ Solutions aux Problèmes de Documentation

Problème : La documentation ScalaDoc ne s'affiche pas correctement

Solutions :

  1. Serveur Local : Utilisez python -m http.server 8080 dans le dossier target/scala-3.3.1/api
  2. Navigateur différent : Essayez Chrome, Firefox, ou Edge
  3. Mode développeur : Ouvrez F12 et vérifiez les erreurs JavaScript
  4. Documentation simplifiée : Utilisez ce fichier HTML pour une vue d'ensemble

Commandes Utiles

# Régénérer la documentation sbt clean sbt doc # Serveur local cd target/scala-3.3.1/api python -m http.server 8080 # Ouvrir dans le navigateur start http://localhost:8080 # Windows open http://localhost:8080 # macOS
